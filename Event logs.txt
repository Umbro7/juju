Системные журналы
	Системные журналы могут быть полезны при устранении неполадок в работе операционной системы . Эти журналы содержат информацию о различных действиях операционной системы.	
- События запуска и завершения работы системы
- События загрузки драйверов
- События системных ошибок
- События, связанные с оборудованием

Журналы безопасности
	Журналы безопасности помогают выявлять и расследовать инциденты. Эти журналы предоставляют информацию о действиях, связанных с безопасностью в системе.	
- События аутентификации
- События авторизации
- События изменения политики безопасности
- События изменения учетной записи пользователя 
- События аномальной активности

Журналы приложений
	Журналы приложения содержат конкретные события, связанные с приложением. Любая интерактивная или неинтерактивная активность внутри приложения будет регистрироваться здесь.	
- События взаимодействия с пользователем
- События изменений в приложении
- События обновления приложения
- События ошибок приложения

Журналы аудита
	Журналы аудита предоставляют подробную информацию об изменениях в системе и событиях, происходящих с пользователями. Эти журналы полезны для соблюдения нормативных требований и могут играть важную роль в мониторинге безопасности.	
- События доступа к данным
- События изменения системы
- События активности пользователей
- События применения политик

Сетевые журналы
	Журналы сети предоставляют информацию об исходящем и входящем трафике сети. Они играют решающую роль в устранении неполадок в сети, а также могут быть полезны при расследовании инцидентов.	- События входящего сетевого трафика
- События исходящего сетевого трафика
- Журналы сетевых подключений 
- Журналы сетевого брандмауэра

Журналы доступа
	Журналы доступа предоставляют подробную информацию о доступе к различным ресурсам. Эти ресурсы могут быть разных типов, что позволяет нам получить информацию об использовании этих ресурсов.
- Журналы доступа к веб-серверу
- Журналы доступа к базе данных 
- Журналы доступа к приложениям
- Журналы доступа к API

cat Команда `cat` — популярная утилита для отображения содержимого текстового файла. Мы можем использовать команду `cat` для отображения содержимого файла журнала, поскольку оно обычно имеет текстовый формат.

использование команды cat

root@kali$ cat access.log
172.16.0.1 - - [06/Jun/2024:13:58:44] "GET /products HTTP/1.1" 404 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
10.0.0.1 - - [06/Jun/2024:13:57:44] "GET / HTTP/1.1" 404 "-" "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_12_3) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
192.168.1.1 - - [06/Jun/2024:13:56:44] "GET /about HTTP/1.1" 500 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"

В большинстве систем журналы регулярно обновляются. Это позволяет создавать отдельные файлы журналов для определенных временных промежутков, а не хранить их все в одном файле. Но иногда может потребоваться объединить два файла журналов. В этом случае может пригодиться утилита командной строки `cat`. Мы можем объединить результаты нескольких файлов в один, как показано ниже.

использование команды cat

cat access1.log access2.log > combined_access.log

grep Это очень полезная утилита командной строки, позволяющая искать строки и шаблоны в файле журнала. Например, вам может потребоваться проверить наличие определенного IP-адреса в вашем файле журнала. Это можно сделать с помощью следующей команды: Следующая команда выполнит поиск строки '192.168.1.1' в файле access.log и отобразит все строки, содержащие эту строку.

Использование команды grep

grep "192.168.1.1" access.log
192.168.1.1 - - [06/Jun/2024:13:56:44] "GET /about HTTP/1.1" 500 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
192.168.1.1 - - [06/Jun/2024:13:53:44] "GET /products HTTP/1.1" 404 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
192.168.1.1 - - [06/Jun/2024:13:46:44] "GET /about HTTP/1.1" 200 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"

Эта less команда полезна для работы с несколькими файлами журналов. Вам может потребоваться анализировать отдельные фрагменты по одному. Для этого можно использовать утилиту командной строки less, которая позволяет просматривать страницы по одной.

less access.log
172.16.0.1 - - [06/Jun/2024:13:52:44] "GET /products HTTP/1.1" 404 "-" "Mozilla/5.0 (Windows NT 6.1; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
10.0.0.1 - - [06/Jun/2024:13:48:44] "GET /about HTTP/1.1" 404 "-" "Mozilla/5.0 (Windows NT 6.1; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"
192.168.1.1 - - [06/Jun/2024:13:46:44] "GET /about HTTP/1.1" 200 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36"

Используйте spacebarдля перехода на следующую страницу и b на предыдущую страницу.

После выполнения этой команды журналы отобразятся постранично, один за другим, что упростит ручной анализ. Если вы хотите выполнить поиск чего-либо в журналах, вы можете ввести текст, /за которым следует искомый шаблон, и нажать Enter. 

Используйте n для перехода к следующему результату поиска и N для перехода к предыдущему результату поиска.
