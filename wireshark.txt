2. Фильтры для поиска SQL-инъекций

Ключевой принцип:

Ищем подозрительные строки в HTTP

http contains "SELECT"
http contains "UNION"
http contains "OR 1=1"
http contains "information_schema"
http contains "SLEEP"
http contains "admin'"

Более точечно:
http.request.method == "POST"

http.request.uri contains "login"

http contains "' OR"
http contains "--"
http contains "#"

Комбинации (очень мощно)
http && ip.addr == 192.168.1.10

http.request.method == "POST" && http contains "OR"

http contains "UNION" && http contains "SELECT"

=================

Statistics --> Resolved Addresses
Разрешенные адреса Эта опция помогает аналитикам идентифицировать IP-адреса иDNSИмена, доступные в файле захвата, можно получить, предоставив список разрешенных адресов и их имен хостов.

Statistics --> Protocol Hierarchy
Иерархия протоколов
Эта опция отображает все доступные протоколы из файла захвата и помогает аналитикам просматривать протоколы в древовидной форме на основе счетчиков пакетов и процентов

Statistic --> Conversations
Беседы Диалог представляет собой трафик между двумя конкретными конечными точками


Statistics --> Endpoints
Конечные точки Опция «Конечные точки» аналогична опции «Разговоры». Единственное отличие заключается в том, что эта опция предоставляет уникальную информацию для одного поля (Ethernet, IPv4, IPv6,TCPиУДПТаким образом, аналитики могут идентифицировать уникальные конечные точки в файле захвата и использовать его для интересующего события. Для просмотра этой информации можно использовать меню

Statistics --> IPvX Statistics
IPv4 и IPv6
До этого момента почти все варианты предоставляли информацию, содержащую обе версии IP-адресов


Statistics --> DNS
Этот вариант разбивает все DNS извлекает пакеты из файла захвата и помогает аналитикам просматривать результаты в древовидной форме на основе счетчиков пакетов и процентного соотношения

Statistics --> HTTP
Этот вариант разбивает всеHTTPизвлекает пакеты из файла захвата и помогает аналитикам просматривать результаты в древовидной форме на основе счетчиков пакетов и процентного соотношения


Английский	C-подобный	Описание	Пример

еq	        ==		Равный		ip.src == 10.10.10.100

ne		!=		Не равны	ip.src != 10.10.10.100

gt		>		Больше чем	ip.ttl > 250

lt		<		Меньше, чем	ip.ttl < 10

ge		>=		Больше или равноip.ttl >= 0xFA

le		<=		Меньше или равноip.ttl <= 0xA


and		&&		Логическое И	(ip.src == 10.10.10.100) AND (ip.src == 10.10.10.111)

or		||		Логическое ИЛИ	(ip.src == 10.10.10.100) OR (ip.src == 10.10.10.111)

not		!		Логическое НЕ	!(ip.src == 10.10.10.222)


Фильтр	Описание
	
http	Показать все HTTP пакеты	

dns	Показать все DNS пакеты

http.response.code == 200 	Показать все пакеты с HTTP код ответа "200"	

dns.flags.response == 0		Показать все DNS запросы

http.request.method == "GET"	Показать все HTTP GET-запросы	

dns.flags.response == 1		Показать все DNS ответы

http.request.method == "POST"	Показать все HTTP POST-запросы	

dns.qry.type == 1		Показать все DNS записи "А"

http.server contains "Apache"	Перечислите все HTTP пакеты, в которых поле "сервер" пакета содержит "Apache" ключевое слово.

http.host matches "\.(php|html)" Перечислите все  HTTP пакеты, в которых поля "host" соответствуют ключевым словам ".php" или ".html".
